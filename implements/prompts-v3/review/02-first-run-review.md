# 首次复盘（项目启动）提示词（v3.0 - 最佳实践版）

## 提示词内容

```
你是项目顾问，从零开始理解项目结构，生成清晰的项目概览。

## 上下文

项目文件统计：
- 总文件数：{{TOTAL_FILES}}
- 文件夹数：{{FOLDER_COUNT}}

## 任务

基于以下项目结构生成项目启动概览。

## 约束

**结构要求**（按顺序）：
1. 项目概览（150 字内：项目类型、主要功能、技术栈、规模）
2. 主要模块（按功能模块组织，每个模块包含功能描述 + 3-5 条代表性文件）
3. 后续关注（3-5 条建议重点区域）
4. 项目文件结构（按文件夹组织，每个文件夹列前 5 个文件）
5. 完整文件列表（所有文件的完整路径，按字母排序）

**识别原则**：
- 按功能模块组织，而非文件类型
- 模块功能描述 2-3 句话
- 文件说明只列出代表性文件，简要说明功能

## 示例结构

```markdown
# 2026-01-14 复盘 - 项目启动

## 项目概览

这是一个基于 React + TypeScript 的 Web 应用，任务管理平台。包含前端 UI 组件、API 客户端、状态管理等模块，属于中小型项目。

## 主要模块

### 用户认证模块
提供用户登录、注册、JWT 令牌管理等认证功能。
- login.tsx - 登录页面组件
- register.tsx - 注册页面组件
- auth.ts - JWT 令牌管理

### UI 组件模块
包含所有可复用的 UI 组件。
- Button.tsx - 通用按钮组件
- Input.tsx - 表单输入组件

## 后续关注

- 建议优先了解 UI 组件模块，这是项目的核心
- 注意 config/ 目录，包含项目的环境配置

## 项目文件结构

### src/components/ (15 个文件)
- Button.tsx
- Input.tsx
...

### src/utils/ (8 个文件)
- logger.ts
- validator.ts
...

## 完整文件列表

- src/components/Button.tsx
- src/components/Input.tsx
...
```

## 输入数据

### 目录结构
{{DIRECTORY_STRUCTURE}}

### 完整文件列表
{{FILE_LIST}}

直接输出 Markdown 格式的复盘，不要添加任何说明。
```

---

## 变量说明
- `{{TOTAL_FILES}}`: 总文件数
- `{{FOLDER_COUNT}}`: 文件夹数
- `{{DIRECTORY_STRUCTURE}}`: 项目目录结构
- `{{FILE_LIST}}`: 完整文件路径列表

## API 建议
- Model: deepseek-v3.2
- Temperature: 0.5
- Max Tokens: 3000

- 长度从 ~4500 字符压缩到 ~900 字符（-80%）
- 移除模块识别方法、质量检查清单等冗余信息
- 将识别原则内化到约束中
- 只保留 1 个示例结构
- 简化输出格式要求
