## 任务

为主题 "**{{TOPIC_TITLE}}**" 撰写 800-1500 字的问题解决方案报告。

## 写作要求

**风格要求**：
- 用自然流畅的语言，避免机械结构
- 最多使用一层标题（##），用 **粗体** 来区分小节
- 案例融入正文叙述，不要用列表格式
- 文章要能从头读到尾，像在分析一个真实问题

**内容要求**：
- 问题背景：问题的上下文和约束条件，用户可能遇到的具体场景
- 解决方案对比：至少 3 种方案的对比（用表格：方案-核心思路-优点-缺点-适用场景）
- 推荐方案：明确推荐一个方案 + 3 条具体理由
- 实施步骤：至少 3 个具体可执行的步骤
- 可能遇到的问题：至少 3 个常见问题及解决方法

**格式要求**：
- 使用 Markdown 格式
- 对比表格格式：| 方案 | 核心思路 | 优点 | 缺点 | 适用场景 |
- 支持 Obsidian 链接语法 [[笔记标题]]

**重要约束**：
- 不要在文章末尾追加任何询问或建议
- 直接输出文章内容，不要有开场白或结束语
